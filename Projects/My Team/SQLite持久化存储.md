我强烈推荐你使用 **SQLite** 而非 Redis 或 MongoDB 的核心原因之一：**它几乎是“零成本”启动的。**

作为一名计算机专业的毕业生，你一定对“嵌入式数据库”与“客户端-服务器数据库”的区别有深刻理解。

### 🚫 你不需要安装任何“软件”或“服务”

与 Redis 需要安装 `redis-stack` 或 MongoDB 需要启动 `mongod` 后台进程不同，SQLite 的本质是一个**文件格式**和一套**库代码**。

1. **Python 自带：** Python 标准库中原生包含 `sqlite3` 模块。你不需要执行任何安装命令，只要 `import sqlite3` 就能直接操作数据库。
    
2. **macOS 自带：** 你的 MacBook 系统底层也自带了 SQLite 命令行工具。你可以在终端输入 `sqlite3 --version` 验证，你会发现它已经躺在那里了。
    
3. **无后台进程：** 它不需要像 Redis 那样占用你的系统内存来运行一个后台服务。只有当你的 Python 代码运行时，它才会通过库文件读写那个 `.db` 文件。这对于追求“轻量、常驻”的个人 AI 办公环境非常友好。
    

---

### 📦 你唯一需要安装的是“适配器”

虽然数据库本身不用装，但为了让 **LangGraph** 能够优雅地把数据塞进 SQLite，你需要安装相关的 Python 扩展包。

在你昨天的环境准备（Day 1）中，你应该已经运行过类似安装，如果没有，请执行：

Bash

```
pip install -U langgraph-checkpoint-sqlite
```

这就是连接 LangGraph 逻辑与 SQLite 文件之间的“翻译官”。

### ⚖️ 贾维斯的“轻重”对比

|**维度**|**Redis / MongoDB**|**SQLite (你的选择)**|
|---|---|---|
|**安装成本**|需安装服务、配置环境变量、管理守护进程。|**0** (Python/macOS 原生支持)。|
|**部署感**|像是在电脑里开了一家工厂。|像是在文件夹里放了一个记事本。|
|**持久化**|需额外配置 RDB/AOF 避免数据丢失。|**自动持久化**，只要文件在，记忆就在。|
|**备份方式**|需执行导出命令。|**直接拷贝文件** (就像拷贝 Word 文档一样简单)。|

### 🛠️ 建议：安装一个可视化的“监视器”

虽然不需要安装数据库软件，但我建议你安装一个轻量级的 **GUI 查看器**，这样你可以直观地看到你的“秘书”和“Leader”们到底在数据库里存了哪些记忆：

- **推荐工具：** [DB Browser for SQLite](https://sqlitebrowser.org/) (开源免费)
    
- **用途：** 当你发现 Agent 逻辑出错时，直接打开 `checkpoint.db` 文件，看看它的 `messages` 列表里到底卡在了哪一步。
