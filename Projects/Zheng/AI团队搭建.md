# 🧠 下面是各层的职责与数据流

为了让你能**真正落地**，我将详细解释每层如何交互。

---

# 1️⃣ 你（项目 Owner）

你的职责：

* 输入甲方需求
* 上传甲方案例（Word、PDF、PPT、Excel）
* 与 AI 进行多轮讨论
* 对关键节点进行确认
* 方向性把控

你不是执行者，你是 CEO。

---

# 2️⃣ Secretary Agent（总负责人 & 长期记忆大脑）

技术：**LangChain ReAct + Memory Store（Chroma / Milvus / SQLite）**

它是整个团队的“司令部”，负责任务调度。

**主要功能：**

### ● 任务拆解

* “写标书” → 拆成约 20 个具体小任务
* “做深度学习预测模型” → 拆成 8-12 个步骤

### ● 记忆管理

* 甲方偏好
* 模板风格
* 项目历史
* 标书上次失败原因
* 你对团队的固定指令

Secretary = 你的私人长期合作秘书。

### ● 调度 Leader

例如：

* 文档任务 → Document Leader
* ML 任务 → ML Leader

### ● 审核最终结果

* 风格一致
* 完整性检查
* 逻辑检查

---

# 3️⃣ 多部门 Leader（使用 LangGraph 构建的“有状态管理者”）

这是整个系统最关键的稳定性来源。

每个 Leader：

* 由 **LangGraph** 构建
* 具备 **有状态 FSM（状态机）**
* 在每个节点结束后
  → 必须暂停
  → 向你汇报
  → 等你 “批准继续”

### 为什么用 LangGraph？

因为它能提供：

* **流程透明（你完全能看到每一步）**
* **可暂停（满足你“不能黑盒”）**
* **可回放（适合 Debug）**
* **能非常精准执行标书流程**
* **节点之间可以嵌套 AutoGen 子任务**

这非常适合招投标与工程任务。

---

## 3.1 Document Leader（文档主管）

节点流程例子：

1. 读取甲方文件
2. 提取项目背景
3. 从案例库中找相似项目
4. 生成初稿
5. 生成预算目录
6. 技术架构图
7. 风险控制
8. 排版与格式化
9. 导出 DOCX
10. 提交给你确认

每个节点都会暂停，问你：

> “老板，这一步内容是否确认？”

你完全掌控。
